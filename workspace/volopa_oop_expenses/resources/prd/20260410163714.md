## Programming Language

Laravel 10+, PHP 8.2+, MySQL

## Original Requirements

Build the Volopa OOP (Out-of-Pocket) Expenses API System using Laravel 10+ with PHP 8.2+. Three modules: 1. User Management & Permissions — RBAC via user_feature_permission with delegation hierarchy 2. Pocket Expense CSV Batch Upload — synchronous validation, async background sync 3. Single Expense Data Capturing — CRUD with FX conversion and flexible metadata

## Product Goals

- Enable comprehensive out-of-pocket expense tracking outside the Volopa card system
- Implement secure role-based access control with delegation capabilities
- Provide efficient CSV batch upload with atomic validation and processing
- Deliver automated FX conversion with user override functionality
- Support flexible metadata capture for expense categorization and tracking
- Ensure seamless integration with existing Volopa platform infrastructure
- Maintain data integrity through proper validation and transaction management
- Enable scalable permission management for multi-tenant client environments

## User Stories

- As a Primary Administrator, I want to enable OOP Expense feature for my client so that all users can start recording out-of-pocket expenses
- As a Primary Administrator, I want to delegate expense management rights to other users so I can distribute workload
- As an Administrator, I want to manage expenses for delegated users so I can assist with expense processing
- As a Business User, I want to create individual expense records with automatic FX conversion so I can track personal business expenses
- As a Card User, I want to upload receipt files with my expenses so I can provide documentation
- As an Administrator, I want to upload CSV files with up to 200 expense records so I can efficiently process bulk expenses
- As a user, I want to see detailed validation errors when my CSV upload fails so I can correct the data
- As a user, I want to configure custom expense sources for my client so I can categorize expenses accurately
- As a user, I want the system to automatically convert foreign currency expenses to my wallet base currency
- As a user, I want to override automatic FX conversion when I have better exchange rate information
- As a Primary Administrator, I want to revoke management rights from users so I can maintain proper access control
- As a user, I want to receive notifications when batch uploads are completed so I know when processing is finished
- As a user, I want to view and edit my own expenses with proper status tracking
- As an Administrator with management rights, I want to approve expenses for my managed users
- As a user, I want to attach flexible metadata like categories, tracking codes, and projects to my expenses

## Requirement Analysis

The Volopa OOP Expenses system requires a sophisticated multi-module Laravel API that integrates seamlessly with existing platform infrastructure. The system must handle three distinct but interconnected concerns: 1) A flexible RBAC system using user_feature_permission table that supports delegation hierarchies beyond traditional role boundaries, 2) A robust CSV batch upload system that provides synchronous validation with detailed error feedback while processing asynchronously in the background, and 3) A comprehensive single expense CRUD system with real-time FX conversion, flexible metadata capture, and proper audit trails. Key technical challenges include maintaining atomic transactions across multiple related tables, implementing proper multi-tenant data isolation, integrating with existing OAuth2 middleware and platform services, and ensuring proper permission enforcement at both the API and database levels. The system must also handle edge cases like FX rate unavailability, CSV validation failures, and complex permission inheritance patterns while maintaining high performance and data integrity standards.

## UI Design draft

The system will integrate with existing Volopa Web App UI through Settings > Manage Users for permission delegation. New API endpoints will support: 1) Expense management dashboard with list/grid view of expenses, status indicators, and filtering capabilities, 2) Single expense form with auto-complete fields for merchant, currency dropdown with FX conversion preview, file upload for receipts, and metadata sections for categories/tracking codes/projects, 3) CSV upload interface with drag-drop file upload, downloadable template, real-time validation progress, and detailed error reporting with downloadable error CSV, 4) Permission management interface integrated into existing user management flow with delegation hierarchy visualization. All APIs will return consistent JSON responses with proper status codes, pagination for list endpoints, and structured error formats for validation failures.
